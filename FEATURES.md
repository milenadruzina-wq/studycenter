# Функциональность системы

## Роли пользователей

### Администратор
Администратор имеет полный доступ к системе и может:

1. **Управление пользователями**
   - Просматривать всех пользователей
   - Изменять роли пользователей (студент/преподаватель/администратор)
   - Управлять активностью пользователей

2. **Управление курсами**
   - Создавать новые курсы
   - Редактировать существующие курсы
   - Удалять курсы
   - Обновлять цены курсов
   - Назначать преподавателей на курсы

3. **Управление группами**
   - Создавать группы
   - Редактировать группы
   - Устанавливать максимальное количество студентов
   - Устанавливать даты начала и окончания

4. **Управление расписанием**
   - Создавать расписание занятий
   - Редактировать расписание
   - Назначать дни недели, время и аудитории

5. **Отчёты**
   - Просматривать статистику по пользователям
   - Просматривать статистику по курсам и группам

**Доступ:** `/admin`

### Преподаватель
Преподаватель может:

1. **Управление посещаемостью**
   - Отмечать посещаемость студентов
   - Выбирать статус: присутствовал, отсутствовал, опоздал, уважительная причина
   - Добавлять примечания к записям посещаемости

2. **Просмотр статистики**
   - Видеть график посещаемости группы
   - Просматривать процент посещаемости
   - Анализировать посещаемость по датам

3. **Выбор группы и даты**
   - Выбирать группу для работы
   - Выбирать дату занятия

**Доступ:** `/teacher`

### Студент
Студент может:

1. **Просмотр курсов**
   - Просматривать доступные курсы
   - Видеть описание, цены, преподавателей
   - Записываться на курсы

2. **Личный кабинет**
   - Просматривать свой профиль
   - Видеть свои записи на курсы
   - Просматривать историю платежей

**Доступ:** `/account`

## API Endpoints

### Посещаемость (требуется аутентификация, доступно преподавателям и админам)

- `GET /api/attendances` - Получить все записи посещаемости
  - Query параметры: `groupId`, `studentId`, `date`
- `GET /api/attendances/stats` - Получить статистику посещаемости
  - Query параметры: `groupId`, `studentId`, `startDate`, `endDate`
- `GET /api/attendances/:id` - Получить запись по ID
- `POST /api/attendances` - Создать запись посещаемости
- `PUT /api/attendances/:id` - Обновить запись
- `DELETE /api/attendances/:id` - Удалить запись (только админ)

## Примеры использования

### Создание записи посещаемости

```bash
POST /api/attendances
Authorization: Bearer <token>
Content-Type: application/json

{
  "studentId": 1,
  "groupId": 1,
  "date": "2024-01-15",
  "status": "present",
  "notes": "Присутствовал на занятии"
}
```

### Получение статистики посещаемости группы

```bash
GET /api/attendances/stats?groupId=1
Authorization: Bearer <token>
```

Ответ:
```json
{
  "total": 50,
  "present": 40,
  "absent": 8,
  "late": 2,
  "excused": 0,
  "attendanceRate": "84.00"
}
```

## Статусы посещаемости

- `present` - Присутствовал
- `absent` - Отсутствовал
- `late` - Опоздал
- `excused` - Уважительная причина
